Why it exists为什么做这个

Three ways SSH managers betray youSSH 管理工具的三种背叛方式

cloud clients云客户端

Your vault on someone else's servers你的保险库在别人的服务器上

Subscription GUIs sync credentials through their cloud. Cancel, get breached, or go offline — you don't own your vault.订阅制 GUI 把凭据同步到它们的云端。停订、被入侵或断网时，保险库根本不属于你。

sshpass

Passwords in files and history密码散落在文件和历史里

Wrapper scripts leave secrets in dotfiles, environment variables, and shell history — readable by anything on your machine.包装脚本把秘密留在配置文件、环境变量和 shell 历史中，机器上任何程序都读得到。

AI + raw sshAI + 裸 ssh

Secrets in the model's context秘密进了模型上下文

Hand an agent your ssh setup and every password it touches lands in its context window and logs. MCP SSH servers have shipped real key-exfiltration and shell-injection CVEs.把 ssh 直接交给智能体，它碰到的每个密码都会进上下文和日志。MCP SSH 服务器已出过真实的密钥外泄与命令注入 CVE。

Security model安全模型

The secret never crosses the line秘密永远不越过这条线

serverctl never types your password. OpenSSH asks a tiny AskPass helper, which matches the prompt to a credential and pulls the value straight from the OS vault into the ssh process — and refuses to answer host-key or OTP prompts that belong to a human. serverctl 从不替你「输入」密码。OpenSSH 向一个极小的 AskPass 助手发问，助手将提示匹配到对应凭据，把秘密从系统保险库直接送进 ssh 进程——并拒绝回答本应由人确认的主机密钥和动态口令提示。

Left of the red line, secrets simply don't exist: not in argv, not in env vars, not in logs, not in any output a model reads. Unsafe plaintext keyring backends are rejected as a hard error. Host-key verification is never weakened — there is no flag for it. 红线左侧不存在任何秘密：参数里没有、环境变量里没有、日志里没有、模型读到的输出里也没有。明文 keyring 后端会被直接报错拒绝。主机密钥校验永不弱化——连绕过它的选项都不存在。

Loopback-only web UI仅本机回环的 Web UI

Binds 127.0.0.1 with a single-use launch token, HttpOnly sessions, CSRF headers, Origin checks, and a strict CSP. The network never sees it.只绑定 127.0.0.1：一次性启动令牌、HttpOnly 会话、CSRF 头、Origin 校验、严格 CSP。网络上根本看不到它。

WebAuthn-gated reveal通行密钥门禁的秘密查看

Displaying a stored secret takes Touch ID / Windows Hello (or an Argon2id master password). Grants are single-use, expire in 30 s, and the value wipes from the page in 15 s.查看已存秘密需 Touch ID / Windows Hello（或 Argon2id 主密码）。授权单次有效、30 秒过期，页面上的值 15 秒后自动清除。

Your ssh config, untouched不碰你的 ssh 配置

Managed hosts render to a separate file, written atomically with user-only permissions; your original ~/.ssh/config is included last and never edited.托管主机渲染到独立文件，原子写入、仅本用户可读；原有 ~/.ssh/config 最后加载，永不被改写。

Features功能

One CLI, a local UI, zero ceremony一个 CLI，一个本地 UI，零繁文缛节

Profiles & jump chains连接档案与跳板链

Aliases with host, port, user, notes, and ordered ProxyJump hops — cycle detection included.别名承载主机、端口、用户、备注和有序 ProxyJump 跳板，自带环路检测。

Three credential kinds三种凭据类型

Vault passwords, private keys with vaulted passphrases, or plain ssh-agent. Reusable across servers, undeletable while referenced.保险库密码、带保险库口令的私钥、或 ssh-agent。可跨服务器复用，被引用时禁止删除。

OpenSSH importOpenSSH 导入

Parses ~/.ssh/config (following Include), resolves aliases with ssh -G, previews before applying.解析 ~/.ssh/config（跟随 Include），用 ssh -G 解析别名，先预览后应用。

Diagnosable tests可诊断的连接测试

Latency plus a classified error code — authentication-failed, host-key-untrusted, timeout, dns-failed — with history.延迟加分类错误码——认证失败、主机密钥不受信、超时、DNS 失败——并记录历史。

Remote execution远程执行

Safely quoted args, --shell for pipelines, stdin streaming for scripts and binaries, connection reuse, JSON results.参数安全转义，--shell 跑管道，stdin 流式传脚本和二进制，连接复用，JSON 结果。

Doctor built in内置体检

serverctl doctor verifies ssh, vault backend safety, paths, and dependencies on any platform.serverctl doctor 在任意平台检查 ssh、保险库后端安全性、路径与依赖。

For AI agents面向 AI 智能体

Agent-drivable. Not agent-readable.智能体驱动得了，读不到秘密。

Ships as an Agent Skill (the open SKILL.md standard supported by Claude Code, Codex CLI, Gemini CLI, Cursor and more). The agent gets a deterministic CLI with --json everywhere and meaningful exit codes; authentication happens out-of-band, so there is no secret for a prompt injection to steal. 以 Agent Skill 形式发布（开放的 SKILL.md 标准，Claude Code、Codex CLI、Gemini CLI、Cursor 等均支持）。智能体得到的是全量 --json 输出和有意义退出码的确定性 CLI；认证在带外完成，提示注入根本偷不到不存在的东西。

  • Structured inventory, tests, and exec results for reliable automation结构化的主机清单、测试与执行结果，自动化更可靠
  • AskPass refuses host-key and OTP prompts — humans stay in the loopAskPass 拒答主机密钥与动态口令提示——人始终在环内
  • Launch tokens and secrets are withheld from stdout by design启动令牌与秘密从设计上就不进入标准输出
